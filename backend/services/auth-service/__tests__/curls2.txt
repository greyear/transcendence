# ========== TOKEN PASSING REFERENCE ==========

# Token Passing Methods:
# 1. HEADER: -H 'Authorization: Bearer <TOKEN>'
#    Use for: Direct token passing, microservice communication
#    Example: curl -H 'Authorization: Bearer <TOKEN>' http://localhost:3000/auth/validate

# 2. COOKIE JAR: -c <jar_file> to save, -b <jar_file> to load
#    Use for: Browser-like behavior, cookie lifecycle testing
#    Example save: curl -c /tmp/cookie.txt http://localhost:3000/auth/register
#    Example load: curl -b /tmp/cookie.txt http://localhost:3000/auth/validate
#    Example both: curl -c /tmp/cookie.txt -b /tmp/cookie.txt http://localhost:3000/auth/logout

# Cookie jar behavior:
#    - Server sends Set-Cookie header with httpOnly, secure, sameSite flags
#    - Curl with -c flag saves cookies to jar file
#    - Curl with -b flag sends cookies from jar file
#    - Cookie jar file format: Netscape HTTP Cookie File (standard curl format)
#    - Cookies include expiration time, path, domain restrictions

# ========== GOOGLE OAuth NOTES ==========

# Google ID Token Source:
# - Obtain from: https://developers.google.com/oauthplayground
# - Expires: ~1 hour from generation
# - Needs refresh periodically in tests
# - Contains: iss, aud, email, email_verified, iat, exp claims

# Google User Behavior:
# - First login with new Google account: Returns 201 with full user details
# - Subsequent logins with same account: Returns 200 with token only
# - Deleted Google user re-login: Creates new account (OAuth behavior)
# - User ID regeneration: Each login may get different internal userId
